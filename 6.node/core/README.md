# 全局变量 Buffer

## 简介

- Buffer 让 JavaScript 语言支持操作二进制数据，因为 IO 读写都是二进制数据，所以需要 Buffer 来操作二进制数据

- 流配合管道可以实现数据分片传输，NodeJS 中的 Buffer 一片不占据 V8 的堆内存大小的内存空间
- 内存的使用还是由 Node 来控制，由 V8 的 GC 回收
- 一般配合流使用，充当数据缓冲区，比如文件读写，网络传输

## 创建实例

- alloc: 创建一个指定大小的 Buffer 实例
- allocUnsafe: 创建一个指定大小的 Buffer 实例，不安全
- from: 接收数据，创建 buffer

## 实例方法

- fill: 使用数据填充 buffer
- write: 向写入数据
- toString: 从 buffer 中提取数据
- copy: 拷贝 buffer 中的数据
- subarray、slice(已废弃): 截取 buffer
- indexOf: 获取 buffer 中指定数据的位置

## 静态方法

- isBuffer: 判断是否是 buffer
- concat: 合并 buffer

# FS 模块

前置：文件的权限位、标识符、文件描述符

- 权限：用户对于文件所具备的操作权限，分为 3 个部分：读、写、执行
  数字表示权限：4、2、1，字符表示权限：r、w、x
- flag ：操作符，表示对文件的操作类型,常用的有 r、w、s、+、x、a。分别如下：
  - r：表示可读
  - w：表示可写
  - s：表示同步
  - +：表示执行相反的操作
  - x：表示排他操作
  - a：表示追加操作
- fd 表示操作系统分配给被打开的文件的文件描述符

# 事件循环

## 浏览器下的事件循环

- 从上至下执行所有的同步代码
- 执行过程中将遇到的宏任务与微任务添加至相应的队列
- 同步代码执行完毕后，执行满足条件的微任务回调
- 微任务队列执行完毕后执行所有满足需求的宏任务队回调
- 循环事件环操作
- 注意：每执行一个宏任务之后就会立刻检查微任务队列，如果有任务就执行，没有任务就继续执行下一个宏任务

## Nodejs 中的事件循环

队列说明：

- `timers` 队列：setTimeout、setInterval
- pending callbacks：执行系统操作的回调，例如 tcp,udp
- idle, prepare：nodejs 内部使用，一般无需关注
- `poll`:执行与 IO 相关的回调
- `check：执行` setImmediate 中的回调
- close callbacks：执行 close 事件的回调

完整事件循环

- 执行同步代码，将不同的任务添加至相应的队列
- 将所有同步代码执行后会去执行满足条件的微任务
- 所有微任务代码执行后会执行 timer 队列中满足的宏任务
- timer 中的所有宏任务执行完毕后，就会以此切换队列
- 注意：在完成队列切换之前会先清空微任务代码

# 流

文件操作系统和网络模块实现了流接口

## 优势

- 时间效率：流的分段处理可以同时操作多个数据 chunk
- 空间效率：数据分段处理，可以避免一次性将数据加载到内存中
- 使用方便：流配合管道，扩展程序变的简单

## 流的分类

- Readable：可读流，用于读取数据
- Writable：可写流，用于写入数据
- Duplex：双工流，既可读又可写
- Transform：转换流，用于处理数据

# 网络通信

必要条件

- 主机之间需要有传输介质如网线，蓝牙等
- 主机上必须要有网卡设备。
- 主机之间需要协商网络速率

## 网络通信方式

- 交换机通讯：
  - 通过交换机将多个主机连接在一起，就形成了一个局域网。
  - 通过 MAC 地址来区分主机。
  - 交换机的数量数量是有限制的，局域网存在大量主机会造成广播风暴
- 路由器通讯
  - 通过路由器将不同的局域网连接在一起。

## OSI 7 层模型

- 应用层：用户与网络的接口:http、ftp
- 表示层：数据加密、转换、压缩
- 会话层：控制网络连接建立和终止
- 传输层：控制数据传输的可靠性
- 网络层：确定目标网络
- 数据链路层：确定目标主机
- 物理层：各种物理设备和标准

## TCP/IP 4 层模型

- 应用层（应用程序协议）
- 传输层（源端口 ↔️ 目的端口）
- 网络层（主机 ↔️ 主机）
- 网络接口层（主机 ↔️ 网络层）

## TCP 三次握手和四次挥手

- 三次握手：客户端发送 SYN，服务端收到后发送 SYN ACK，客户端收到后发送 ACK，完成三次握手
- 四次挥手： 客户端发送结束信号(FIN)； 服务端回应确认(ACK)并可能发送剩余数据；服务端发送结束信号(FIN)；客户端确认服务端结束信号(ACK)，双方等待规定时间后关闭连接。

## 创建 TCP 通信

- 创建服务端：接收和回写客户端数据
- 创建客户端：发送和接收服务端数据
- 数据传输：内置服务事件和方法读取数据

- net 模块通信事件 & 方法
  - listening 事件：调用 server.listening 方法后触发。
  - connection 事件：新的连接建立时触发
  - close 事件：当 server 关闭时触发
  - error 事件：当错误出现时触发。
  - data 事件：当接收到数据的时候触发该事件。
  - write 方法：在 socket 上发送数据，默认是 UTF8 编码
  - end 方法：当 socket 的一端发送 FIN 包时触发，结束客户端

封包就是给一段数据加上包头，这样一来数据包就分为包头和包体两部分内容了（可加上包尾）。包头其实是一个大小固定的结构体，其中有个结构体成员变量表示包体的长度，这是个很重要的变量，其他的结构体成员可根据需要自己定义。根据固定的包头长度以及包头中含有的包体长度变量值就能正确的拆分出一个完整的数据包。
